# 海上风电历史数据分析与多算法智能体系统

基于 2022-2023 年福建海上风电历史数据，集成功率建模验证、风险量化评估、多策略调度对比与智能问答的一体化分析平台。

## 核心理念

本系统定位为**历史数据分析与模型评估平台**，而非实时预测系统：

- **数据时间线**：训练集(2022-01 ~ 2023-04 前80%) → 验证集(2023-01 ~ 2023-04 后20%，有真实功率) → 推演期(2023-05 ~ 2023-07，无真实功率)
- **精度验证**：在验证集上对比模型预测功率与真实功率，用 MAE/RMSE/MAPE 量化评估
- **模型推演**：在竞赛A榜测试集上展示模型输出，但不声称"预测"（无真实值可对比）

## 技术架构

```text
数据层 → 特征工程(38维) → 双模型预测(HistGBR/LightGBM) → 风险评分(7因素)
                                                          → RL调度(QL/DQN/GA/启发式)
→ Flask API → Vue 3 前端大屏
```

## 核心模块

| 模块 | 功能 | 亮点 |
|------|------|------|
| `data_utils.py` | 数据加载、清洗、38维特征工程 | 时序安全划分、循环编码、风向量分解 |
| `forecasting.py` | 风电功率建模 (HistGBR + LightGBM) | 双模型对比评估、风险评分系统 |
| `rl_control.py` | 调度策略优化 (Q-Learning + DQN + GA + 启发式) | 6种策略统一对比、Double DQN + 软更新 |
| `system.py` | 系统编排 | 验证集精度分析、自动优选最优模型 |
| `ollama_client.py` | 本地 LLM 问答 | 自动检测 Ollama 服务，回退规则问答 |
| `app.py` | Flask API 后端 | RESTful 接口，CORS 支持 |

## 运行方式

### 1. 训练系统

```powershell
python train_system.py
```

训练完成后会自动：
- 保存结果到 `artifacts/runs/run_<timestamp>/`
- 比较新旧结果，更新 `artifacts/best_run.txt`

### 2. 启动后端

```powershell
python app.py
```

后端默认地址: `http://127.0.0.1:5000`

### 3. 启动前端

```powershell
cd frontend
npm install
npm run dev
```

前端默认地址: `http://127.0.0.1:5173`

## 前端页面

| 页面 | 路由 | 内容 |
|------|------|------|
| 总览大屏 | `/overview` | 验证集精度分析、数据时间线、风险热区、策略对比 |
| 站点驾驶舱 | `/sites` | 地图、推演曲线、日汇总、风险窗口 |
| 算法实验页 | `/experiments` | 多策略对比矩阵、自定义算法入口 |
| 智能体中心 | `/agent` | 问答工作台、快捷提问、数据导出 |

## Ollama 集成

如果本地运行了 Ollama 服务 (`http://127.0.0.1:11434`)，系统会自动使用 LLM 进行智能问答。
优先模型: `deepseek-r1:7b` → `llama3:latest`。
不可用时自动回退到内置中文规则问答。

## 演讲稿建议

1. 从数据时间线开始，说明训练/验证/推演的划分逻辑
2. 重点展示**验证集预测 vs 实际对比**，证明模型精度
3. 说明推演期(5-7月)仅为模型输出，无法验证
4. 展示多算法调度对比，突出 Q-Learning 和 DQN 的差异
5. 用高风险窗口说明不同站点需要不同策略
6. 以智能问答收尾，展示系统的综合性
